Campus Quest: The Mid-Term Mission

Make choices. Shape your destiny. Reach one of 10 endings.

Addendum (Process & Reflection)

Planning: I mapped the story as a flowchart with one start node, several mid-nodes, and 10 distinct ending nodes. Each ending includes a unique image and short outcome title for clarity.

Structure: I separated concerns: HTML for structure, CSS for layout/theme, and JavaScript for logic. The story is stored as a dictionary of nodes. Each node has text, image, and choices (label + next key). Ending nodes also include isEnding and endingTitle.

Interactivity: The page updates via DOM manipulation in renderNode(). Buttons are created dynamically with event listeners. endGame() removes choices and shows a restart option.

Debugging: I tested reachability for all endings, validated relative image paths (images/...), and confirmed buttons route to valid node keys. I used the browser console to catch typos or missing keys.

Enhancements: I added responsive layout, accessible landmarks, focus styles, and smooth transitions for a better UX.

Lessons Learned: Designing branches first prevents dead ends. Keeping content in data (not hard-coded in HTML) makes iteration faster and safer.
